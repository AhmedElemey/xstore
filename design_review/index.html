Known issues to review

Please review these first — they are visible in the screenshots below and are already tracked, not new findings.

  1. Vendor — Incoming Orders. The order list area renders empty due to a layout crash (infinite-height constraint). The header stats still render, but no order cards appear. This screenshot captures the real broken state, not a placeholder.
  2. Vendor — My Listings (empty state). The floating “+ New Listing” button overlaps and clips the “Add Your First Listing” call-to-action.
  3. Mock data — market mismatch. Several screens show Algerian cities, addresses, and names (Algiers, Constantine, “12 Rue Didouche Mourad”) in what should be an Egypt-market app. The vendor profile even reads “Algiers, Egypt.”
  4. Consumer — Order Detail. A delivered order shows a “Processing — Pending” step sitting between two already-completed timeline steps.
  5. Notifications — empty state copy. The empty-state text reads “Your all notifications will appear here,” which reads as a grammar error.
  6. Vendor — bottom navigation. The tab label truncates to “Incoming Ord…” at the standard 390px width.
  7. Order Detail — footer overlap. The “Cancel Order” footer button slightly overlaps the scrollable content above it.
